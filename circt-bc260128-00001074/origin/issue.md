## Description

`circt-verilog` crashes while lowering a Moore SV module that declares a `string`-typed port. The crash occurs in `hw::ModulePortInfo::sanitizeInOut()` during MooreToCore conversion, likely because a `string` port type is converted to an invalid/empty HW type and then unconditionally `dyn_cast`ed as `hw::InOutType`.

**Crash Type**: assertion (original) / segfault (current tool)
**Dialect**: Moore
**Failing Pass**: MooreToCore

## Steps to Reproduce

1. Save the test case below as `test.sv`
2. Run:
   ```bash
   circt-verilog --ir-hw test.sv | arcilator | opt -O0 | llc -O0 --filetype=obj -o test.o
   ```


## Root Cause Analysis

`getModulePortInfo()` converts port types using the MooreToCore `TypeConverter` without validating the result. When the `string` port type conversion yields an invalid/empty type, `hw::ModulePortInfo` construction calls `sanitizeInOut()`, which unconditionally `dyn_cast`s each port type to `hw::InOutType`. The dyn_cast assertion fires on the invalid type.
